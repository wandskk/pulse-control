# 05 - UI/UX

## Diretriz principal

A aplicação deve ser **mobile first**, simples, objetiva e com foco em acionamento rápido.

## Stack visual

- Tailwind CSS
- shadcn/ui
- layout limpo
- áreas de toque confortáveis
- feedback visual imediato

## Telas do MVP

### 1. Home / Dashboard
Elementos:
- cabeçalho simples
- seletor de dispositivo
- botão “Adicionar número”
- grid/lista de comandos
- botão “Novo botão”
- acesso ao histórico

### 2. Modal de dispositivo
Campos:
- nome
- telefone
- descrição opcional

### 3. Modal de comando
Campos:
- título
- mensagem SMS
- cor predefinida

### 4. Histórico
Elementos:
- lista de execuções
- data/hora
- botão executado
- status
- mensagem resumida

## Diretrizes de interação

- ações principais sempre visíveis
- modais curtos
- formulário com poucos campos
- toasts para sucesso e erro
- loading no botão acionado
- confirmação para exclusão

## Diretrizes visuais

- botão deve ter contraste adequado
- a cor do comando deve ser consistente em card e CTA
- evitar tela poluída
- preferir 1 coluna no mobile
- no tablet/desktop, evoluir para grid

## Cores de comando sugeridas

- azul
- verde
- vermelho
- amarelo
- violeta
- cinza escuro
